## Introducción
**¿Qué es GitHub y cómo funciona?**

Git es un sistema que te permite gestionar y seguir los cambios en uno o varios archivos, como el código de un proyecto. Cualquier miembro del equipo con acceso puede ver el historial de cambios, modificar el código o revertir cambios si es necesario.

En Git, se pueden crear "ramas" o versiones paralelas del código. Cada desarrollador puede trabajar en su propia rama de forma independiente sin afectar el código principal, que está en la rama maestra. Cuando los cambios están listos, se pueden unir de nuevo a la rama maestra para actualizar el proyecto con esos cambios.

**¿Qué se verá en este informe?**

En este informe explicaremos qué es GitHub, para qué se utiliza y detallaremos los pasos necesarios para realizar diferentes acciones dentro de la plataforma.

## Pasos a seguir para conseguir las diferentes acciones

1. Crear un repositorio.
   
   Empiezas un proyecto creando un repositorio, donde guardarás y organizarás tu código.
   
2. Hacer commits.

   Un commit es como un guardado de los cambios que hiciste en el código. Después de modificar algo, usas el comando _git 
commit_ para guardar esos cambios junto con un mensaje que explique qué hiciste.

3. Creación de ramas.

   Cuando estés en tu nuevo repositorio, haz clic en el botón **main**, escribe el nombre de tu nueva rama y luego pulsa **Create branch**. Así, habrás creado una copia exacta de la rama principal (maestra).

4. Hacer push y pull.

   Después de hacer commits, usas _git push_ para subir los cambios al repositorio remoto, y con _git pull_ puedes descargar las actualizaciones más recientes para asegurarte de que tu trabajo esté sincronizado con la versión actual del proyecto.
   
6. Hacer un Pull Request.

   Cuando comiences a colaborar con otros usuarios, es el momento de solicitar su revisión. Esto les permite comentar o sugerir modificaciones en tu propuesta de cambios antes de fusionarla con la rama principal (main).

## Conclusiones
En resumen, GitHub es una plataforma esencial para la gestión de proyectos de código, ya que facilita la colaboración entre los desarrolladores. Permite crear repositorios para organizar el código, hacer commits para registrar cambios y utilizar ramas para trabajar de forma independiente. Las funciones de push y pull aseguran que todos los miembros del equipo puedan acceder a la versión más reciente del proyecto. Además, el uso de Pull Requests facilita la revisión y mejora del código antes de integrarlo en la rama principal, asegurando un flujo de trabajo organizado y efectivo. De esta forma, GitHub mejora tanto la calidad del software como la colaboración efectiva.
